## Sprint 1 — Верстка с заглушками и fixed photo naming

**Статус: СЛЕДУЮЩИЙ**

### Цель

Собрать весь интерфейс поэтапно на заглушках, с готовностью к замене фото через uploads.

### Общие правила спринта

- Реальные фото не блокируют верстку: использовать заглушки и fixed file names.
- Работать только в production-файлах (`index.html`, `css/*`, `js/*`).
- Не копировать код из `prototype/`.

### Подэтапы (отдельные таски)

#### 1.1 Header + Footer

- sticky header, burger/drawer, CTA «Забронировать», телефоны.
- footer с навигацией и соцссылками.

**Acceptance:**
- корректная адаптивность desktop/mobile;
- CTA переводит к `#booking-widget`;
- клик по логотипу: на `/` — scroll to top; с других страниц — переход на `/` + scroll to top (§SPEC 4.3).

#### 1.2 Главная

- PageHeader;
- 4 шахматных блока;
- CTA-баннер;
- секция бронирования (`#booking-widget` + `#agast-widget-container` placeholder).

**Acceptance:**
- блоки соответствуют структуре и текстам из данных;
- на мобиле корректный порядок и сетка.

#### 1.3 Разводящая `/rooms`

- сетка карточек;
- бейдж площади, цена, ссылка в детальную.

**Acceptance:**
- карточки рендерятся из `js/data.js`;
- переход в `/rooms/:id` работает без перезагрузки;
- CTA на карточке ведёт к бронированию **этой** категории в Agast (заглушка до Sprint 3).

#### 1.4 Деталки `/rooms/:id`
- 8 деталок
- галерея (main + превью);
- характеристики, удобства, цена, CTA.

**Acceptance:**
- переключение превью работает;
- CTA «Забронировать» открывает бронирование **этого** номера в Agast (заглушка до Sprint 3).

#### 1.5 Страница прайса `/price`

- шаблон страницы по контракту из `SPEC.md`;
- таблица и блоки условий.

**Acceptance:**
- роут `/price` доступен;
- контент может быть временно marked as pending до финальных тарифов.

#### 1.6 Политика ПД + cookies

- страница `/privacy` под физлицо;
- cookie banner и сохранение consent.

**Acceptance:**
- баннер появляется для новых пользователей;
- сохраняется выбор accepted/declined.

### Зависимости

- утвержденная структура страницы `/price`.

### Риски

- задержка финальных тарифов;
- отсутствие финальных фото.

### Задачи

- Проверка `robots.txt` и `sitemap.xml`.
- Smoke-test основных пользовательских сценариев.

### Acceptance

- Сайт доступен по `https://abrikos-yeisk.ru`;
- маршруты и интеграции работают на production.

### Риски

- ошибки rewrite-правил на хостинге;
- кэширование старых файлов браузером/CDN.
